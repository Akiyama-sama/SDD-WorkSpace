# Proposal

## 目标

把口语化需求转成结构化提案，至少说清楚：

- 要做什么
- 为什么做
- 什么算完成
- 不做什么
- 有哪些显式约束

## 最小流程

1. 识别是否已有活跃变更。
2. 没有活跃变更时创建新的 `openspec/changes/<change>/proposal.md`。
3. 已有活跃变更时，判断这是续写还是变更范围修订。
4. 产出简洁提案，避免过早进入实现细节。

## 提案模板

```md
# <变更标题>

## 背景

## 目标

## 完成标准

## 非目标

## 约束与依赖
```
