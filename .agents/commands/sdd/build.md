# Build

## 目标

从规格进入实现，并确保每次修改都能回到明确的任务与验证标准。

## 流程

1. 找到活跃变更及其 `tasks.md`。
2. 如存在 `impl-plan.md`，优先断点恢复。
3. 没有实现计划时，根据 `tasks.md` 生成最小执行计划。
4. 逐项执行任务，完成后立即回写状态。
5. 每一批改动后运行对应验证。

## 约束

- 不要跳过未完成任务直接宣称“已完成”。
- 不要脱离规格自行扩 scope。
- 如果实现中发现规格有问题，先回写规格再继续。
