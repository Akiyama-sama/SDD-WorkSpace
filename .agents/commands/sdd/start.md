# Start

## 目标

在不依赖长会话上下文的前提下，决定本次需求应该：

- 直接进入轻量实现。
- 先走需求收敛与规格生成。
- 从已有变更断点恢复。

## 步骤

1. 检查 `.agents/sdd-state.json` 是否存在。
2. 检查 `openspec/changes/` 中是否有活跃变更。
3. 如存在活跃变更，判断当前处于：
   - 仅有 `proposal.md`
   - 已有 `design.md` / `tasks.md`
   - 已进入实现
   - 已准备关闭
4. 评估本次需求适合的档位：
   - `vibe`：非常小且低风险
   - `plan`：中等复杂度，先写规格
   - `full`：高风险、多模块或需要长期追踪
5. 将档位与当前阶段写入 `.agents/sdd-state.json`。

## 输出

- 对当前状态的判断
- 建议档位
- 建议下一步命令
