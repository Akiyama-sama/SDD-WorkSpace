# 工程上下文

> 这是通用模板，请按你的项目实际情况补全。建议让这里始终保持“AI 初次进入仓库时必须知道的最小集合”。

## 1. 项目定位

- 这是一个使用 SDD 方法组织研发流程的工作区。
- 它可以服务于单仓项目、多仓项目，或一个聚合式研发工作区。
- 产物以 Markdown 为主，代码仓可按需挂载或并排放置。

## 2. 典型目录职责

| 路径 | 作用 |
|------|------|
| `openspec/changes/` | 活跃变更与归档 |
| `openspec/specs/` | 稳定规格沉淀 |
| `vertical-knowledge/rules/` | 通用规则与工程约束 |
| `vertical-knowledge/wiki/` | 领域知识索引 |
| `.agents/commands/` | 命令工作流说明 |
| `.agents/tools/` | 可选自动化脚本 |
| `dashboard/` | 查看变更产物的示例前端 |

## 3. 你应该在这里补充什么

- 当前项目使用的技术栈与版本。
- 常用启动、测试、构建命令。
- 关键目录和边界说明。
- 哪些模块风险高，改动前需要额外验证。

## 4. 推荐写法

- 用事实描述当前项目，不写抽象口号。
- 优先写“怎么找到入口”和“怎么验证改动”。
- 如果一条信息已经过时，就更新或删掉，不保留历史垃圾。
