# AI 行为规范

## Ingest

1. 先读取当前变更的 `proposal.md`、`design.md`、`tasks.md` 与相关 `specs/`。
2. 仅在知识库无法回答时，再继续探索代码、日志或外部文档。
3. 更新知识库时，只修改与当前变更直接相关的最小知识单元。
4. 变更写回前，给出“新增 / 修改 / 废弃”的摘要，便于人工确认。
5. 不直接改动与当前任务无关的知识条目。

## Query

1. 从 `index.md` 找到相关模块或主题入口。
2. 先读规则层，再按需进入 wiki 层。
3. 回答时标注结论来自哪一类知识：规则、架构、模块、历史变更。

## Lint

1. 检查是否存在互相矛盾的知识条目。
2. 检查是否存在没有任何入口引用的孤儿页。
3. 检查近期变更是否已同步回长期知识。
4. 输出修复建议，优先人工确认后再执行。
